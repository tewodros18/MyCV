Pixels Vs Relative units in CSS styling, Does it matter?

Yes! The most important reason for using responsive and relative values in our CSS is for supporting users that rely on scaling the content for reasons including various visual inablities including myopia ,dyslxia. Users need to be able to zoom the viewport without loss of content or functionality, or restrictions imposed by CSS values or viewport scaling settings.

  • must be able to resize text without assistive technology up to 200 percent, without loss of content or functionality.

  • must be able to resize text without being forced to scroll both horizontally and vertically to read that content.

The main counter argument that can be posed for using fixed values like pixels instead of relative values like rem or em is the continuous improvment in the ablitiy of browsers to provide Zoom functionality. Since most popular browsers offer the option to zoom a given webpage why should web developers go the extra mile to calculate and change pixels into relative values? Well as long as the browser's size is set to the default value often 16px the Zoom function is adquate to navigate but The problem arises when the WebPage is visted by a user using a browser with a different default font size. If a user changes the default settings in a browser, ideally all the fonts will scale up (or down) proportionately in reference to the size of the font selected in the setting. Ideally, that is, if the webpage is using relative CSS units. This browser font size adjustment won’t have any effect on typography that uses fixed value like pixels for font-size and/or the line-height. this failure to scale up proportionally will often cause static fonts and compressed line spacing, and such line spacing can be tough to parse for users who have a cognitive disability

In addition, This will force the user to continuously zoom in and out as they navigate from one sectioni of the website to the other breaking the seamless interaction that can be easily acheived by using the relative values.

In conclusion, when designing a website developers should consider the fact that users do change their default font settings, and the developer should be maintaining users’ control over their own browsing experience. If relative CSS units are used for the typography of the website it maintains the fidelity of the WebPage's layouts without negatively impacting the needs of users.

"16px", how does a browser's default width and fixed values work

Absolute length units such as pixels are fixed in relation to each other.

  • in: inches — 1in is equal to 2.54cm.
  • pt: points — the points used by CSS are equal to 1/72nd of 1in.
  • px: pixel units — 1px is equal to 0.75pt.

For a CSS device, these dimensions are either anchored (i) by relating the physical units to their physical measurements, or (ii) by relating the pixel unit to the reference pixel. For print media and similar high-resolution devices, the anchor unit should be one of the standard physical units (inches, centimeters, etc). For lower-resolution devices, and devices with unusual viewing distances, it is recommended instead that the anchor unit be the pixel unit. For such devices it is recommended that the pixel unit refer to the whole number of device pixels that best approximates the reference pixel. The reference pixel is the visual angle of one pixel on a device with a pixel density of 96dpi and a distance from the reader of an arm's length. For a nominal arm's length of 28 inches, the visual angle is therefore about 0.0213 degrees. For reading at arm's length, 1px thus corresponds to about 0.26 mm (1/96 inch).

So after understanding What 1px represents next thing is what happens when a browser renders an HTML file i.e a webpage its going to display it in the default browser font size which is 16px so any scaling to be done such as using 200px will be done proportionally taking the 16px or 1rem as a base line.
